  NEXT_PUBLIC_API_URL=http://localhost:8000

FastAPI Backend Requirements:
  Endpoint: POST /api/auth/login
  Request: { email, password }
  Response: { access_token, token_type, user: { id, email, name, role } }

================================================================================
KEY FEATURES
================================================================================

1. Authentication Flow:
   - User enters email/password on login page
   - Credentials sent to NextAuth provider
   - Provider calls FastAPI backend /api/auth/login
   - Backend validates and returns user with JWT token
   - NextAuth creates session with user role
   - User redirected to dashboard

2. Session Management:
   - JWT-based sessions (not database-backed)
   - 24-hour session expiration
   - User role included in session
   - Automatic token refresh within session lifetime
   - Secure HTTP-only cookies

3. Route Protection:
   - Middleware protects /dashboard routes
   - Unauthenticated users redirected to /login
   - Role-based access control available
   - Admin-only routes can be restricted

4. Type Safety:
   - Full TypeScript support
   - Extended NextAuth types for user role
   - Typed API responses
   - Typed backend authentication responses

5. Security:
   - CSRF protection (automatic)
   - HTTP-only cookies
   - Session encryption (NEXTAUTH_SECRET)
   - Strong error handling
   - Role-based access control
